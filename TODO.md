## 1. Уже заявленное в README (перенести сюда и детализировать)

- 🔴 **`--scene N`** — перегенерировать один кадр без пересбора всего батча. Нужно: параметр принимает int или список (`--scene 5,12,33`), сбрасывает у этих сцен `status=pending` и `image_path=""`, затем обычный цикл. Полезно для точечных правок.
- 🟡 **Параллельная генерация картинок** — `nano-banana` stateless, можно гнать N сцен одновременно через `asyncio` + семафор (лимит ~5, чтобы не ловить 429). Ожидаемое ускорение на 128 сценах: ×3–5.
- 🟡 **Context caching для dossier + story** — Gemini поддерживает explicit cache, снимает до 75% стоимости на повторяющейся части prompt'а. Для batch это не критично, но для realtime-режима даст реальную экономию.
- 🟡 **`--render-video`** — финальная ffmpeg-склейка: кадры + voice.mp3 + SRT + design_spec → mp4. Сейчас пользователь делает это вручную.
- 🟢 **Авто-подбор голоса per-speaker** — интеграция с TTS-бэкендом (ElevenLabs / Yandex SpeechKit / Silero), чтобы `speaker: narrator` и персонажи звучали разными голосами автоматически.

## 2. TTS-интеграция (самый большой пробел в пайплайне)

Сейчас пользователь должен сам гонять `voice_text` через внешний TTS. Это половина всей ручной работы.

- ✅ **`--tts`** — флаг генерации через ElevenLabs (апрель 2026). Yandex/Silero — следующие итерации.
- ✅ **Маппинг speaker → voice_id** в `voices.json` с fallback-цепочкой `speaker → default → narrator`.
- ✅ **Сохранение аудио** в `output/audio/scene_NNN.mp3` с атомарной записью.
- ✅ **Кэш TTS** по hash(text + voice_id + model + settings) — per-scene в progress.json.
- 🟡 **Учёт emotion/pacing** в настройках TTS — не закрыт (текущая реализация берёт static settings).
- 🟡 **Нормализация аудио** (ffmpeg loudnorm) — не закрыт (ждёт `--render-video`).
- 🟢 **Yandex SpeechKit** — следующий подпроект, потребует provider abstraction.
- 🟢 **Silero** — локальный fallback, следующий подпроект.

## 3. Видеосборка (`--render-video`)

- ✅ **Базовая склейка** (2026-04-22): каждая сцена → клип `effective_duration` с картинкой + аудио + hard-sub ASS. Concat demuxer, quality presets draft/final, hash-кэш.
- ✅ **Burn-in субтитров** (2026-04-22): ASS-формат из `design_spec.json` (font/color/stroke/position), `subtitle_lines` с `\N` line-break.
- 🟡 **Ken Burns эффект** (медленный pan/zoom) на статичных кадрах — убирает ощущение слайдшоу. Можно выбирать направление случайно или на основе композиции.
- 🟡 **Crossfade** между сценами (0.3–0.5с) вместо жёсткой склейки.
- 🟢 **Бэкграунд-музыка**: `--bgm path/to/track.mp3` with auto-ducking (понижение громкости музыки во время речи).
- 🟢 **Разные соотношения сторон**: генерация 9:16 варианта для вертикального видео (TikTok/Reels/Shorts) — отдельный `STYLE_SUFFIX` с safe zones по центру.

## 4. Качество консистентности персонажей

Главная боль nano-banana: при 4+ персонажах в кадре лица плывут.

- 🔴 **Несколько ракурсов для reference-портрета** — сейчас один frontal. Делать 3 ракурса (front, 3/4, profile) и передавать все → качество сильно растёт.
- 🔴 **Эмоциональные варианты reference** — базовый neutral + 2–3 ключевых эмоции (angry, sad, scared) как отдельные картинки, LLM выбирает подходящий по `emotion` сцены.
- 🟡 **Consistency check** — после генерации кадра отдельный vision-call «на картинке реально {aleksey_38}?» с reference-сравнением. Если нет — ретрай с усиленным промптом.
- 🟡 **Хранение одежды по эпохам** — в `characters.json` добавить поле `wardrobe_by_year`, чтобы Алексей-2003 был в подростковой одежде, Алексей-2020 — в казённой робе. Сейчас LLM пытается это инферить каждый раз.
- 🟢 **Pose-reference** — передавать схематичный скетч позы как дополнительный ref (ControlNet-подход, но через multimodal prompt).

## 5. Улучшения scene splitter

- 🔴 **Настраиваемая плотность сцен** — флаг `--target-scenes N` или `--density low|medium|high`. Сейчас LLM сам решает, может выдать 80 или 180 сцен на том же тексте.
- 🟡 **Ручная правка split** — команда `--edit-split` открывает `progress.json` в `$EDITOR` до запуска prompt-билда, чтобы можно было объединить/разрезать сцены.
- 🟡 **Валидация split**: каждая сцена должна быть подстрокой story (verbatim). Сейчас проверки нет — если LLM галлюцинирует, никто не заметит.
- 🟢 **Визуальный preview split** — вывод в консоль: номер, длина, первые слова, персонажи. Перед дорогим запуском.

## 6. Кост-контроль и телеметрия

- 🔴 **Dry-cost estimator** — команда `--estimate`, без API-вызовов прикидывает стоимость: количество сцен × средний промпт × цена токена по актуальной таблице. Сейчас пользователь узнаёт цифру постфактум.
- 🔴 **Usage tracking**: записывать в `output/usage.json` количество input/output токенов и стоимость по каждому вызову. После запуска — сводка.
- 🟡 **Soft budget limit** — `--max-cost 10` прерывает работу, если оценка общей стоимости превысила лимит.
- 🟡 **Per-scene cost в `prompts.json`** — видно, какие сцены дорогие (обычно из-за эскалации на Pro).

## 7. CLI / UX

- 🔴 **Progress bar** для длинных операций (batch polling, генерация картинок). Сейчас — только `log.info`, в длинных ранах теряется ощущение жизни.
- 🟡 **`--dry-run-html`** — после dry-run генерировать HTML-превью: все сцены списком, промпт, voice_text, субтитры. Правят в браузере до дорогой картиночной генерации.
- 🟡 **Статус-команда** — `python generate_comic.py --status` показывает по `progress.json`: сколько ok/skipped/error, какие сцены ждут batch, сколько стоило.
- 🟡 **Интерактивный режим правки** — `--review`: после каждой картинки показ превью (через imagemagick/PIL) и спрашивает «ok / retry / skip». Для ручной отладки.
- 🟢 **Web UI** на FastAPI + минимальный frontend: drag-n-drop `story.txt`, preview кадров, точечный re-roll. Самый удобный способ для не-девелоперов.

## 8. Надёжность и тестирование

- ✅ **Unit-тесты** для чистых функций: `classify_error`, `backoff_delay`, `_fmt_srt_time`, `estimate_duration`, `pick_scene_model` — покрыты в `tests/test_pure.py` (апрель 2026).
- ✅ **Schema validation** ответов LLM через `pydantic` — все 4 LLM-вызова (split, bootstrap, scene prompts, design spec) валидируются, retry-on-ValidationError встроен в `call_llm_json`. Batch path помечает сцену `status=error` при fail. Реализовано апрель 2026.
- 🟡 **Integration-тест** на замоканном `genai.Client`: прогнать 3-сценный story через весь пайплайн, проверить структуру `prompts.json` и `subtitles.srt`.
- 🟡 **Golden test** для SRT: зафиксировать эталонный выход на тестовом story, регрессии ловятся сравнением.
- 🟡 **Retry budget с предохранителем** — если сумма ретраев по всем сценам превысила X, останавливаться. Иначе зависший эндпоинт может сожрать сотни вызовов.
- 🟢 **Chaos-тесты** — прогон с моком, который случайно возвращает 503/429/битый JSON, проверка что пайплайн не падает.

## 9. Рефакторинг кода

Файл `generate_comic.py` ~1050 строк — для персонального скрипта терпимо, но если хочется растить фичи, стоит разбить.

- 🟡 **Разделить на модули**:
  ```
  comic/
    __init__.py
    cli.py              # argparse + main
    llm.py              # call_llm_json, classify_error, backoff
    scenes.py           # split_story, build_scene_prompt
    batch.py            # batch_create/collect
    images.py           # generate_image
    bootstrap.py        # characters, references, design_spec
    subtitles.py        # estimate_duration, export_srt
    models.py           # Scene dataclass
    config.py           # константы, модели
  ```
- 🟡 **Вынести промпты в `prompts/*.md`** — сейчас огромные f-строки в коде. Markdown-файлы легче редактировать и ревьюить.
- 🟡 **Конфиг-файл `config.toml`** — модели, пороги, стили, лимиты. Сейчас всё константами в коде, не поменяешь без правки `.py`.
- 🟢 **Async-клиент** — `genai` поддерживает `async`. Переписать hot-path (генерация картинок) на `asyncio.gather`.

## 10. Стилистика и вариативность

- 🟡 **Несколько стилевых пресетов** — не только «Soviet comic-noir». Добавить `--style cyberpunk|ghibli|watercolor|sovietnoir`, каждый со своим `STYLE_SUFFIX` и `PORTRAIT_STYLE_SUFFIX`. Держать в `styles/*.toml`.
- 🟡 **Per-scene style override** — некоторые сцены в флешбэках хорошо бы рендерить иначе (выцветший, сепия). Поле `style_override` в Scene.
- 🟢 **Aspect ratio переключатель** — `--aspect 16:9|9:16|1:1|4:5` с соответствующими safe-zone инструкциями в промпте.

## 11. Работа с `characters.json`

- 🟡 **Merge-стратегия при bootstrap** — сейчас `--bootstrap` без `--force` пропускает существующие id, но не обновляет description у изменённых. Добавить `--bootstrap-merge`, который докидывает новые поля к существующим.
- 🟡 **Валидация id**: snake_case, ascii, уникальность. Если LLM придумал `Алексей-38`, скрипт должен отловить.
- 🟡 **Возрастные переходы** — сейчас `lyosha_5, lyosha_8, ...` задаются вручную. Автогенерация возрастных варианов по всем годам, упомянутым в story.
- 🟢 **Character-relationship граф** — кто кому родственник/коллега/антагонист. LLM мог бы использовать для composition («X стоит напротив Y с дистанцией»).

## 12. Субтитры и локализация

- 🟡 **Multi-lang SRT** — `--subtitle-langs ru,en`. Второй проход LLM делает перевод `voice_text` → английский, пишет `subtitles.en.srt`.
- 🟡 **ASS формат** — богаче SRT (стили, позиционирование, эффекты), поддерживает то, что в `design_spec.json` (stroke, gradient). Генерировать параллельно.
- 🟡 **Проверка длины строк** — сейчас `SUBTITLE_MAX_CHARS_PER_LINE=42` только в документации. Валидатор должен реально обрезать/разбивать строки, вышедшие за лимит.
- 🟢 **Синхронизация по словам** — если TTS возвращает word-level timestamps (Whisper-форматирование), пересчитывать SRT с точностью до слова, а не сцены.

## 13. Наблюдаемость и отладка

- 🔴 **Structured logging** — переключатель `--log-format json`, каждая запись с fields (scene_index, model, cost, latency). Намного легче грепать большие раны.
- 🟡 **Per-scene debug dump** — флаг `--debug`, каждый вызов LLM сохраняет в `output/debug/scene_NNN/{prompt.txt, raw_response.json}`. Позволяет понять, почему картинка вышла странной.
- 🟡 **Latency metrics**: минимум, медиана, p95 по каждой модели. В конце ранa печатать таблицу.
- 🟢 **Интеграция с OpenTelemetry** — если проект разрастётся, traces покажут bottleneck.
